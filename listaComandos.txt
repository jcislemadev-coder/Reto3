git commit: Guarda de forma permanente en el repositorio una instantánea de los cambios preparados en el area de montaje (staging area) Da una descripción y los alista para cargarlos. Es algo similar a un punto de guardado esencial que registra el historial permitiendo rastrear evoluciones y volver a versiones anteriores del código
git push: Sube los commits desde la rama local en el repositorio git local al repositorio remoto
git status: Proporciona una lista de los archivos que están sin preparar y los archivos preparados el estado en que se encuentran los archivos, modificados (modified), preparados (staged) sin seguimiento (untracked)
git add: Mueve cambios (archivos nuevos, modificados o eliminados), desde el directorio de trabajo al área de preparación. Es un paso intermedio esencial antes de confirmar (commit), permite seleccionar que cambios específicos de guardaran
